DriftFrame · Unsplash Photos + ChatGPT

Privacy Policy

Effective date: July 20, 2026

DriftFrame is a Chrome new-tab extension. This policy explains the data the extension handles, why it is used, and the controls available to users.

Data handled

  • Unsplash Access Key: Entered by the user and stored only in chrome.storage.local on that device. It is sent directly to the Unsplash API solely to request photographs and report user-initiated downloads as required by the Unsplash API.
  • Wallpaper preferences and favorites: Topic, refresh interval, favorite identifiers, photo URLs, photographer names, and attribution links are stored with chrome.storage.sync. If Chrome Sync is enabled, Google may sync this information between the user’s signed-in Chrome installations.
  • Current wallpaper and cache time: Stored locally to reduce unnecessary API requests.
  • ChatGPT prompt: When the optional ChatGPT handoff is used and the user presses Enter, the submitted prompt is stored locally for no more than two minutes, used to fill the ChatGPT composer, and then deleted. After verifying the text was filled, DriftFrame clicks ChatGPT’s send control once. Pressing Enter in DriftFrame is the user’s confirmation to send that prompt.
  • Downloads: A download begins only after the user clicks a download control and is handled by Chrome’s download system.

Third-party services

DriftFrame communicates directly with Unsplash to request and display photographs. If the user enables the optional ChatGPT handoff, DriftFrame sends only the user-submitted prompt to chatgpt.com. Unsplash, OpenAI, Google, and Chrome process information under their own privacy policies.

What DriftFrame does not do

DriftFrame has no advertising, analytics, tracking, sale of data, or developer-operated backend. It does not collect browsing history, read unrelated pages, or transmit user data to the developer.

User control and deletion

Users can revoke ChatGPT site access from Chrome’s extension details, remove favorites individually, clear extension data in Chrome, or uninstall the extension. Chrome Sync and deletion of synced extension data are managed through the user’s Google account and Chrome settings.

Security and limited use

All third-party requests use HTTPS. Permissions are limited to the features described above. ChatGPT access is optional and requested only when the feature is first used. DriftFrame’s use of information received from Google APIs complies with the Chrome Web Store User Data Policy, including the Limited Use requirements.

隐私政策

生效日期：2026 年 7 月 20 日

DriftFrame 是一款 Chrome 新标签页扩展。本政策说明扩展会处理哪些数据、处理原因以及用户可以使用的控制方式。

处理的数据

  • Unsplash Access Key：由用户输入，只存储在当前设备的 chrome.storage.local 中。它只会被直接发送给 Unsplash API，用于请求照片以及按照 Unsplash API 规范报告用户主动发起的下载。
  • 壁纸设置与收藏：主题、更换间隔、收藏标识、照片网址、摄影师姓名和署名链接存储在 chrome.storage.sync 中。如果用户启用 Chrome 同步，Google 可能在其已登录的 Chrome 设备之间同步这些信息。
  • 当前壁纸与缓存时间：保存在本地，用于减少不必要的 API 请求。
  • ChatGPT 提示词：用户使用可选的 ChatGPT 跳转功能并按下回车后，提示词会在本地最多暂存两分钟，用于填入 ChatGPT 输入框，随后删除。验证文字已填入后，DriftFrame 会点击一次 ChatGPT 的发送按钮。在 DriftFrame 中按回车即表示用户确认发送该提示词。
  • 下载：只有用户主动点击下载按钮时才会开始，并由 Chrome 下载系统处理。

第三方服务

DriftFrame 会直接连接 Unsplash 获取和显示照片。如果用户启用可选的 ChatGPT 跳转功能，扩展只会把用户主动提交的提示词发送到 chatgpt.com。Unsplash、OpenAI、Google 和 Chrome 分别依据其隐私政策处理信息。

DriftFrame 不会做什么

DriftFrame 不包含广告、分析统计、追踪、数据出售或开发者自建后端；不会收集浏览历史、读取无关网页，也不会把用户数据传给开发者。

用户控制与删除

用户可以在 Chrome 扩展详情页撤销 ChatGPT 网站权限、逐项删除收藏、清除扩展数据或卸载扩展。Chrome 同步及同步数据的删除由用户在 Google 账户和 Chrome 设置中管理。

安全与有限使用

所有第三方请求均使用 HTTPS。扩展权限仅用于上述功能。ChatGPT 权限是可选的，首次使用该功能时才会请求。DriftFrame 对从 Google API 获得的信息的使用遵守 Chrome 应用商店用户数据政策，包括“有限使用”要求。
